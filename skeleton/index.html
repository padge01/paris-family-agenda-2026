Paris with Patrick — Piece 1 UX skeleton

Goal

Build a family-facing decision page, not an itinerary worksheet.

The page should help Allison/Laura understand the geography and then click simple preferences. Padraig/Forge can assemble the final itinerary from those preferences.

Core decision model

The family should answer:

1. How hard should we push the trip?

2. What should Monday feel like after the airport bus?

3. What should Tuesday morning prioritize?

4. What should Tuesday afternoon/evening prioritize?

5. What food/café experience sounds best?

The family should NOT have to choose between rigid package plans.

Proposed final page structure

1. Hero / orientation

Purpose: explain the trip in one screen.

Content:

  • Paris with Patrick
  • One arrival afternoon + one full Paris day
  • Monday starts after the Beauvais → Porte Maillot airport bus
  • Airbnb check-in is normally 3:00pm
  • Tuesday is the main Paris day
  • Wednesday is airport morning

Tone:

  • warm
  • simple
  • no mockup/meta language
  • no planning jargon

2. Map-first overview

Purpose: make Paris intuitive before asking for choices.

Map should show:

  • Porte Maillot / A01 airport bus
  • apartment at 36 Rue du Faubourg Saint-Martin
  • Old Catholic Paris cluster: Saint-Nicolas, Notre-Dame, Sainte-Chapelle, St-Etienne
  • Rue du Bac / Saint-Germain cluster: Miraculous Medal Chapel, Saint-Sulpice, Le Bon Marche, classic cafés
  • West Paris cluster: shopping streets, Arc, Trocadero / Eiffel view
  • Optional: Sacre-Coeur / Montmartre

Legend groups:

  • Transit
  • Apartment / base
  • Catholic / old Paris
  • Coffee & pastry
  • Shopping
  • Classic photos
  • Optional

3. Overall pace

Choices:

  • Easy
  • Balanced
  • Ambitious

This should be the first click because it controls everything else.

4. Monday after airport bus

Choices:

  • Old Catholic Paris

- Saint-Nicolas, Notre-Dame area, maybe Sainte-Chapelle or St-Etienne, then apartment after check-in.

  • Gentle arrival

- pastry/coffee, easy walk, settle in, early dinner.

This must reflect that we do not need to drop bags at the Airbnb first and check-in is not until 3pm.

5. Tuesday morning

Choices:

  • Rue du Bac + Saint-Germain

- Miraculous Medal Chapel, Le Bon Marche, Saint-Sulpice, café/pastry.

  • Notre-Dame / Sainte-Chapelle fallback

- protects central Catholic Paris if Monday becomes too tired.

6. Tuesday later day

Choices:

  • Shopping + pretty streets

- Rue Saint-Honore, Rue Cambon, Place Vendome, Avenue Montaigne.

  • Classic photo finish

- Arc de Triomphe, Trocadero / Eiffel view.

  • Montmartre only if energy is good

- Sacre-Coeur / Montmartre as a swap, not an add-on.

7. Food / café / dinner preferences

Choices should be by experience, not a restaurant database:

  • Easy bakery near apartment: Mamiche / Du Pain et des Idees
  • Classic Saint-Germain café: Café de Flore / Les Deux Magots
  • Elegant pastry / hot chocolate: Angelina
  • Eiffel-view café: Carette if Trocadero route
  • Dinner near apartment
  • Dinner where the day ends

Rule: every food place name must be linked at the point it appears.

8. Live preference summary + submit

Purpose: collect clicks without copy-to-chat.

Final version should use a real submit mechanism:

  • Tally / Google Form for lowest risk
  • Google Apps Script to Sheet for custom but simple
  • Supabase for fully live app-like voting

Do not implement final submit until UX is approved.

What to cut from prior versions

Cut:

  • Plan A / B / C / D package voting
  • dense “customize the plan” grid
  • heart buttons on every place
  • long place encyclopedia
  • copy-to-chat as final mechanism
  • internal/meta language like mockup, planning model, anchor, direction, shape, role

Keep:

  • map
  • simple choice cards
  • visible Saint-Nicolas / SSPX anchor
  • Rue du Bac / Notre-Dame / Sainte-Chapelle prework
  • food links where referenced
  • baby-flex framing
  • check-in and airport constraints

Success criteria for the finished page

A family member should understand within 20 seconds:

  • where the airport bus, apartment, Catholic stops, cafés, shopping, and photo stops are
  • that Monday is flexible because check-in is 3pm
  • that Tuesday is the main Paris day
  • what they are supposed to click
  • that they are giving preferences, not locking in a rigid itinerary
